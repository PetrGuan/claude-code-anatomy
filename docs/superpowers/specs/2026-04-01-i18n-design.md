# i18n Internationalization Design Spec

## Overview

Add English/Chinese bilingual support to claude-code-anatomy. English is the default locale (no URL prefix), Chinese uses `/zh/` prefix.

- English: `/overview`, `/query-pipeline`, etc.
- Chinese: `/zh/overview`, `/zh/query-pipeline`, etc.

## Architecture

### Astro i18n Configuration

```js
// astro.config.mjs
i18n: {
  defaultLocale: 'en',
  locales: ['en', 'zh'],
  routing: { prefixDefaultLocale: false }
}
```

### Translation System

**`src/i18n/locales.ts`** — Type definitions and utility functions:
- `Locale` type: `"en" | "zh"`
- `t(locale, key)` — look up translated UI string
- `getLocalePath(locale, path)` — generate locale-prefixed path
- `getAlternatePath(locale, currentPath)` — get the same page in the other language

**`src/i18n/translations.ts`** — All UI strings in both languages:
- Navigation labels
- Page titles, subtitles, section headings
- Plain/technical explanation text for every LayerToggle
- Button labels, footer text, sidebar headers
- Code snippet annotations

### Page Structure — Content Component Pattern

Each page is split into:
1. **Content component** (`src/content/XxxPage.astro`) — contains all layout and content logic, accepts `locale` prop
2. **Thin page files** — one per locale, just passes locale to the content component

```
src/pages/overview.astro          →  <OverviewPage locale="en" />
src/pages/zh/overview.astro       →  <OverviewPage locale="zh" />
```

This eliminates duplication — content logic lives in one place.

### Data Files

Existing data files already have bilingual fields (`name`/`nameCn`, `description`/`descriptionCn`, `label`/`labelCn`). Add helper functions:

```ts
// In each data file or a shared helper
export function localized<T extends { name: string; nameCn: string }>(
  item: T, locale: Locale
): string {
  return locale === "zh" ? item.nameCn : item.name;
}
```

### Component Changes

**Navbar.astro** — Accept `locale` prop. Translate nav labels. Add language switcher (`EN | 中文`) that links to the alternate locale version of the current page.

**BaseLayout.astro** — Accept `locale` prop. Set `<html lang="en">` or `<html lang="zh-CN">`. Pass locale to Navbar and Footer.

**Footer.astro** — Accept `locale` prop. Translate footer text.

**LayerToggle wrapper components** — Each wrapper (LayerToggleScale, LayerToggleQueryWhat, etc.) accepts `locale` prop and renders the appropriate language content.

**Pure display components** (StatCard, FlowDiagram, BubbleChart, TreeExplorer, etc.) — No changes needed. They receive content via props from the page level.

### Language Switcher

Simple `<a>` tag in Navbar, no JavaScript state:
- On `/overview` → shows "中文" linking to `/zh/overview`
- On `/zh/overview` → shows "EN" linking to `/overview`

### What We Don't Do

- No runtime language switching (page navigation on switch)
- No auto-detection of browser language
- No partial translation fallback (every locale is complete)
- No third-party i18n libraries
